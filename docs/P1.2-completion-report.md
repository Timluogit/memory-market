#### 架构设计

```
┌─────────────────┐
│   客户端请求    │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ AuditMiddleware │ ───► 异步写入队列
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│   业务处理      │
└─────────────────┘
         │
         ▼
┌─────────────────┐
│   响应返回      │
└─────────────────┘

后台线程：
┌─────────────────┐
│   队列批量写入  │ ───► 数据库
└─────────────────┘
```

#### 数据库表结构

**AuditLog** - 约15个字段
- 操作者：actor_agent_id, actor_name
- 操作：action_type, action_category
- 目标：target_type, target_id, target_name
- 请求：http_method, endpoint, ip_address, user_agent
- 结果：status, status_code, error_message
- 数据：request_data, response_data, changes
- 安全：session_id, request_id, signature
- 时间：created_at

**AuditLogExport** - 约12个字段
- 导出者：exported_by_agent_id, exported_by_name
- 配置：export_format, filters, record_count
- 状态：status, progress, error_message
- 文件：file_path, file_size, file_url, expires_at
- 时间：created_at, completed_at

#### 性能优化

1. **异步写入**：不阻塞主请求
2. **队列缓冲**：批量减少数据库操作
3. **索引优化**：多维度索引支持高效查询
4. **分页查询**：避免大结果集
5. **缓存统计**：定期更新统计摘要
