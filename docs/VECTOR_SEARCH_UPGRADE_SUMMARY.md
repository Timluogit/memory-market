## 技术方案总结

### 1. 架构设计

```
┌─────────────┐
│  用户查询    │
└──────┬──────┘
       │
       ▼
┌─────────────────────────┐
│  混合搜索引擎             │
│  (HybridSearchEngine)    │
└──────┬──────────────┬───┘
       │              │
       ▼              ▼
┌──────────────┐ ┌──────────────┐
│ 向量搜索     │ │ 关键词搜索   │
│ (Qdrant)     │ │ (SQL LIKE)   │
└──────────────┘ └──────────────┘
       │              │
       └──────┬───────┘
              │
              ▼
       ┌──────────────┐
       │  Rerank 融合  │
       └──────┬───────┘
              │
              ▼
       ┌──────────────┐
       │  排序 + 分页 │
       └──────┬───────┘
              │
              ▼
       ┌──────────────┐
       │  返回结果    │
       └──────────────┘
```

### 2. 核心组件

#### Qdrant 向量数据库

- **Collection:** `memories`
- **向量维度:** 512 (bge-small-zh-v1.5)
- **距离度量:** Cosine
- **索引:** HNSW (m=16, ef_construct=100)
- **量化:** Scalar INT8

#### sentence-transformers

- **模型:** `BAAI/bge-small-zh-v1.5`
- **特点:** 中文优化、速度快、精度高
- **设备:** CPU / GPU / MPS

#### 混合检索策略

```python
# 融合算法
hybrid_score = (
    vector_score * 0.6 +    # 语义权重 60%
    keyword_score * 0.4    # 关键词权重 40%
)

# Rerank 多维排序
rerank_score = (
    base_score * 0.5 +  # 原始搜索分数
    text_sim +          # 文本相似度
    signal_score +      # 信号质量
    time_score +        # 时效性
    price_score         # 价格合理性
)
```

### 3. 性能优化

- **批量索引:** 100 条/批
- **延迟加载:** 按需加载嵌入模型
- **异步向量化:** 后台线程执行
- **HNSW 索引:** 平衡速度和精度
- **量化存储:** INT8 量化，内存优化
