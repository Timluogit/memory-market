## 缓存策略

### TTL（过期时间）

- **默认**: 1小时（3600秒）
- **可配置**: 通过`CACHE_TTL`环境变量
- **推荐范围**: 300秒（5分钟）~ 86400秒（24小时）

### 淘汰策略

- **策略**: `allkeys-lru`
- **内存限制**: 2GB（可通过`CACHE_MAX_MEMORY`配置）
- **行为**: 内存达到上限时，优先淘汰最近最少使用的键

### 失效策略

#### 主动失效

数据更新时立即失效相关缓存：
- 记忆更新 → 失效搜索、用户、记忆缓存
- 团队更新 → 失效团队缓存
- 用户更新 → 失效用户缓存

#### 延迟失效

- **默认延迟**: 5秒
- **目的**: 防止雪崩效应
- **可配置**: 通过`CACHE_DELAY_SECONDS`环境变量

## 故障排查

### 缓存命中率为0

**检查**:
```bash
# 缓存是否启用
curl http://localhost:8000/api/v1/cache/config

# Redis是否连接
curl http://localhost:8000/api/v1/cache/health
```

### 缓存未失效

**解决**:
```python
# 手动清空缓存
import httpx
httpx.delete("http://localhost:8000/api/v1/cache/clear")
```

### Redis内存不足

**解决**:
```bash
# 降低最大内存
# 在 redis/redis.conf 中修改
maxmemory 1gb

# 重启Redis
docker-compose -f docker-compose.cache.yml restart redis
```
