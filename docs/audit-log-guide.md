# 审计日志功能指南

## 概述

审计日志系统用于记录系统中所有关键操作，满足企业合规要求，支持审计追踪和问题排查。

## 功能特性

### 1. 日志记录

**自动记录的操作：**
- 用户登录/登出
- 记忆创建、更新、删除
- 购买和交易
- 团队管理操作
- 系统配置变更

**记录的信息：**
- 操作者信息（ID、名称）
- 操作类型和类别
- 目标对象（类型、ID、名称）
- 请求信息（HTTP方法、端点、IP地址、User-Agent）
- 操作结果（状态、错误信息）
- 详细数据（请求体、响应体、变更内容，已脱敏）

### 2. 数据脱敏

**自动脱敏的敏感信息：**
- 密码：完全脱敏（`***`）
- Token/API Key：完全脱敏（`***`）
- 信用卡号：保留最后4位（`************1234`）
- 社会安全号：保留最后4位（`***-**-1234`）
- 邮箱：保留首字符和域名（`t***@example.com`）
- 电话：保留最后4位（`****1234`）

**脱敏策略：**
- 字段名匹配：根据字段名称判断是否脱敏
- 内容检测：检测字符串内容中的敏感信息
- 递归处理：支持嵌套字典和列表

### 3. 日志查询

**查询条件：**
- 时间范围（开始日期、结束日期）
- 操作类型（create、update、delete、login、export...）
- 操作类别（auth、memory、team、transaction、system）
- 目标对象（类型、ID）
- 操作者（用户ID）
- 操作状态（success、failure、error）
- 关键词搜索

**分页支持：**
- 默认每页20条
- 最大每页100条
- 按时间倒序排列

### 4. 导出功能

**支持的格式：**
- CSV：适合 Excel 查看
- JSON：适合程序处理
- PDF：适合正式报告

**导出流程：**
1. 创建导出任务
2. 后台异步生成文件
3. 生成下载链接（24小时有效）
4. 下载或自动过期清理

**导出限制：**
- 仅管理员可导出
- 支持按过滤条件导出
- 导出文件自动清理（48小时后删除）

### 5. 数据保留策略

**保留策略：**
- 默认保留90天
- 可配置保留天数（最少30天）
- 过期日志自动清理
- 支持归档（可选）

**清理机制：**
- 自动清理过期日志
- 分批删除（避免长时间锁表）
- 定时任务触发（每天凌晨）

## 常见问题

### Q1: 审计日志会影响性能吗？

**A:** 影响很小（<5%），原因：
- 异步写入：不阻塞主请求
- 批量插入：减少数据库操作
- 队列缓冲：平滑写入峰值
- 脱敏优化：快速处理敏感数据

### Q2: 如何确保日志不被篡改？

**A:** 当前版本提供：
- 独立的数据库表
- 严格的权限控制
- 不可删除的记录（硬编码）

**未来版本将增加：**
- 数字签名验证
- 区块链存证
- 只读数据库副本

### Q3: 敏感信息会完全脱敏吗？

**A:** 基本策略：
- 密码、密钥：完全脱敏
- 信用卡、SSN：保留最后4位
- 邮箱、电话：部分脱敏

**注意：** 脱敏不是加密，仍需加强访问控制

### Q4: 导出文件多久过期？

**A:** 当前设置：
- 下载链接：24小时过期
- 导出文件：48小时后自动删除

**建议：** 导出后及时下载并妥善保管

### Q5: 如何自定义脱敏规则？

**A:** 扩展脱敏规则：

```python
from app.core.sanitizer import Sanitizer

# 自定义脱敏器
class CustomSanitizer(Sanitizer):
    def __init__(self):
        super().__init__()
        # 添加自定义敏感字段
        self.SENSITIVE_PATTERNS.extend([
            r'custom_field',
            r'sensitive_data',
        ])
```

## 故障排查

### 问题1：日志记录缺失

**可能原因：**
- 中间件未正确注册
- 请求跳过审计（健康检查等）
- 数据库写入失败

**解决方法：**
- 检查中间件配置
- 查看数据库连接状态
- 检查错误日志

### 问题2：脱敏不生效

**可能原因：**
- 字段名不匹配敏感模式
- 自定义规则未加载
- 缓存问题

**解决方法：**
- 检查字段命名
- 更新脱敏规则
- 重启应用

### 问题3：导出任务卡住

**可能原因：**
- 后台任务未启动
- 数据量过大
- 文件系统权限问题

**解决方法：**
- 检查后台任务状态
- 增加导出批次大小
- 检查文件系统权限
